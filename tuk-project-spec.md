# 툭(TUK) — P를 위한 무계획 기록 앱

## 한 줄 소개
계획 없이 사는 사람(MBTI P)을 위한 앱. 카테고리를 미리 정하지 않고 아무거나 던지면 AI가 자동으로 분류해주고, 월말엔 나무 형태로 회고할 수 있다.

## 핵심 철학
- 계획·목표·스트릭(연속 기록) 없음. 압박감을 주지 않는다.
- 분류는 사용자가 하지 않는다. AI가 알아서 한다.
- 수정은 "해도 되고 안 해도 되는" 선택 사항.
- 사용자를 절대 진단하지 않는다 (번아웃이다, 우울하다 같은 단정 금지).

---

## 핵심 플로우
1. 홈 화면 하나의 입력창에 텍스트/사진/음성으로 아무거나 "던진다"
2. AI가 자동으로 카테고리(가지)와 세부 태그(열매)를 붙인다 (기록당 세부 태그 최대 2개)
3. 홈 피드는 카드 리스트로, 태그는 눌러서 필터링
4. 카드를 탭하면 펼쳐지며 "태그 고치기 / 지우기" 등장 (평소엔 안 보임)
5. "나무" 탭에서 월별 회고. 가지 굵기 = 카테고리 빈도, 열매 크기 = 세부 태그 빈도
6. 좌우로 지난달 나무도 넘겨볼 수 있음

## 태그 체계 (2단계)
**카테고리(가지, 5개 고정)**: 식단 / 감정 / 할일 / 소비 / 관계

**세부 태그(열매, AI가 자유롭게 붙임)**: 카테고리마다 여러 개. 예시:
- 식단: 혼밥, 야식, 카페인
- 감정: 기분좋음, 무기력, 스트레스
- 할일: 할일, 약속
- 소비: 소비, 충동구매
- 관계: 친구

세부 태그는 확장 가능해야 함 (사용자가 반복 사용하는 새 표현을 AI가 새 태그로 제안 → 사용자 승인 시 추가).
기록 하나당 세부 태그는 최대 2개까지만 (지저분해 보이는 것 방지).

## 나무 시각화 로직
- 가지 길이/굵기 = 그 달에 해당 카테고리가 나온 횟수 (많을수록 굵고 김)
- 열매 크기 = 그 세부 태그가 나온 횟수 (비례)
- 안 쓴 카테고리는 잎/열매 없이 앙상한 가지만 (완전히 숨기지 않음)
- 열매를 누르면 "#야식 · 식단 가지 · 이번 달 4번" 같은 상세 정보 카드 표시 + "기록 보기"로 실제 기록 리스트 이동
- 달마다 다른 나무 모양 → 좌우 화살표로 과거 달 넘겨보기
- 빈 달/적은 달은 "이 달은 이렇게 쉬어갔네요" 등 죄책감 없는 톤으로 표현. 비교(지난달보다 적었다 등)는 강조하지 않음

## AI 인사이트 (월간 리포트, 나무 탭 하단)
- 이번 달 총 기록 수, 가장 많이 나온 태그
- 자주 함께 나오는 태그 조합 발견 (예: "#야식과 #무기력이 함께 뜬 날이 3번 있었어요")
- 절대 진단하지 않는 톤: "그냥 그때의 당신 리듬이에요. 좋다 나쁘다 없이요."
- 부정 감정 태그(무기력/스트레스)가 반복(예: 월 4회 이상) 관찰되면, 판단 없이 "혹시 버겁다면 전문가와 이야기해보는 것도 방법이에요" 정도로만 담담하게 안내

## 태그 수정 UX
- 평소 카드에는 수정 버튼이 보이지 않음 (깔끔함 유지)
- 카드의 텍스트 영역을 탭하면 펼쳐지며 "태그 고치기 / 지우기" 등장
- 태그 고치기: 기존 태그 X로 제거, 추천 태그 목록에서 추가 (최대 2개)
- 태그를 다 지우고 "태그 없음(순수 메모)" 상태로 둘 수 있음
- 수정하면 "고쳤어요 · 비슷한 기록은 다음부터 이렇게 붙일게요" 안내 → 개인화 학습에 반영 (그 사용자의 표현-태그 매핑을 우선 적용)
- 태그를 누르면 필터링 / 텍스트를 누르면 펼치기·수정으로 인터랙션 분리

## 온보딩
- 로그인 없이 즉시 입력창 노출, "가입 없이 일단 던져보기"가 메인 CTA
- Google/Apple 원탭 로그인은 보조 옵션
- 가입 전 기록은 기기에 로컬 저장 → 가입 시 그대로 계정에 연결
- 튜토리얼 페이지 없음. 필요한 순간에만 짧은 툴팁

## 알림 (리텐션)
- 하루 최대 1회, 완전히 랜덤한 시간 (새벽 제외)
- "3일째 안 쓰셨어요" 같은 압박형 문구 금지
- 과거 기록을 소환하는 방식: "지난주 이맘때 뭐 던졌는지 기억나요?"
- 최근 자주 쓰면 알림 빈도 자동 감소. 알림을 계속 무시해도 빈도 자동 감소 (스스로 눈치채는 앱)

## 수익 모델
- 기본 무료 + 비침투적 광고 (리포트 열람 후 등 자연스러운 지점에만)
- 인사이트/리포트 기능은 영원히 무료 원칙 (바이럴 공유의 핵심이므로 유료화하지 않음)
- 유료 옵션 후보(추후 검토): 광고 제거, 무제한 보관 기간, 테마/스티커 등 부가 기능
- AI 호출 비용이 사용자 수에 비례해 증가하므로, 베타 단계에서 실제 단가 계산 필요

## 개인정보/신뢰 원칙 (반드시 지킬 것)
- 기록 내용을 광고 타겟팅에 사용하지 않음
- 기록을 제3자에게 판매·제공하지 않음
- 민감 데이터는 기기 우선 저장, 서버엔 최소한만
- 언제든 전체 데이터 내보내기(export) 가능
- 언제든 전체 삭제 가능 (삭제 시 서버에서도 즉시 파기, 실수 방지 위해 확인 단계 필수)
- 만 14세 미만 가입 시 법정대리인 동의 필요
- AI 분류·인사이트는 참고용이며 의료/심리 진단이 아님을 명시

## 위험 요소 (설계 시 주의)
- 익명 커뮤니티/채팅 기능은 넣지 않거나, 넣더라도 사람 간 직접 소통이 아닌 통제된 형태(예: 리포트 이미지 공유)로 제한
- MBTI를 유형 간 편가르기/차별 소재로 소비되지 않게 톤 관리
- 감정 데이터 다루는 부분은 실제 개발 전 심리 전문가 자문 권장

## 화면 목록 (MVP 범위)
1. 온보딩 (가입 없이 시작)
2. 홈 (입력 + 피드 + 태그 필터 + 카드 확장/수정/삭제)
3. 나무 (월 전환 + SVG 나무 + 열매 상세 + AI 인사이트)
4. 설정 (계정, 알림 토글, 개인정보 약속, 데이터 내보내기/삭제, 약관)
5. 이용약관 / 개인정보 처리방침

## 기술적으로 필요한 것 (Claude Code 단계에서 구현)
- 프론트엔드: 웹(React) 우선 → 이후 모바일 앱화(React Native 등) 검토
- 백엔드: 사용자 인증, 기록 저장 DB, AI 분류 API 연동(LLM 호출)
- AI 분류: 사용자 텍스트를 받아 카테고리+세부태그(최대 2개)를 반환하는 프롬프트 설계, 사용자별 수정 이력을 반영하는 개인화 로직
- 알림: 로컬 푸시 스케줄링 (매일 랜덤 시간 1회 예약)
- 데이터 내보내기: JSON/텍스트로 전체 기록 다운로드
- 앱스토어 배포 전 심사 대응(개인정보 처리방침 URL, 연령 등급 등) 별도 준비 필요

# [추가] 소비 브랜치 확장 스펙 (구 "결" 기능 통합)

## 배경 / 정체성 재정의
- 별도 앱 "결"(소비+감정 기록)을 만들지 않고 툭의 소비 브랜치로 흡수.
- 툭 = "일기 앱"이 아니라 "하루 중 스쳐가는 것들을 던지면 AI가 정리하는 라이프로그". 소비/감정 기록은 그 안의 한 조각이지, 가계부 앱이 되는 게 아님.

## 우선순위 원칙 (중요)
새 기능 검토 시 항상 먼저 판단: **"새로운 입력 방식을 요구하는가, 아니면 기존 데이터를 다르게 보여주는 뷰(view)인가?"**
- 뷰 추가 = 저비용, 계속 확장 가능
- 새 입력 방식(계좌 연동, 일정 등록 등) = 별도 프로젝트 수준 비용. 신중히.

## 추가 사항
1. **소비 fruit 서브태그에 감정 계열 추가** (최대 2개 제한 유지)
   - 필요 (연한 클레이 #FAECE7)
   - 스트레스 (중간 클레이 #F0997B)
   - 충동 (진한 클레이 #993C1D)
   - 톤온톤: 감정이 진할수록 색이 진해짐. 무지개색 아님.
2. **월간 리포트 소비 브랜치에 "색 캘린더" 토글 뷰 추가**
   - 기존 나무 시각화는 유지. 토글로 캘린더 뷰 전환.
   - 월 그리드, 하루 칸이 그날 소비 감정 색으로 채워짐
   - 기록 없는 날은 빈칸(연회색), 페널티성 표시 없음
   - 범례: 필요/스트레스/충동
3. **스트릭 카드 (소형 위젯)**
   - "충동소비 없는 날" 연속 카운트. 저금통 아이콘이 카운트에 비례해 차오름.
   - 별도 화면 아님. 색 캘린더 아래 붙는 위젯.
4. **주간 리마인드 알림 톤**
   - 지출 재촉 금지. 회고형. "얼마 쓰셨어요?"(X) → "오늘 하루 어떠셨어요?"(O)
5. **월말 공유 카드**
   - 소비 색 캘린더를 9:16 이미지로 자동 생성. 텍스트 최소화, 색 패턴 중심. 인스타 스토리 공유 유도.

## 하지 않을 것 (의도적 제외)
- 별도 "결" 앱/브랜드 (이름·로고·독립 온보딩 모두 폐기)
- 특정 종목/상품 매수·매도 제안 (규제 이슈)
- 은행/증권사 계좌 자동 연동 (마이데이터 라이선스 필요)
- 진짜 일정관리형 캘린더 (알림·동기화·초대). 캘린더는 "이미 기록된 소비를 색으로 보여주는" 읽기 전용 뷰로만 한정
- 리퍼럴 보상, 사용자 간 랭킹/비교 (차분한 톤과 불일치, 스트레스성 소비 유발 우려)

# [추가] 브랜드 정체성 & 보이스 (핵심 승부수)

## 한 줄 정체성
**"안 써도 괜찮은 기록 앱"** — 다이어리·가계부·습관 앱을 깔았다 관둔 사람들("실패한 기록자")을 위한 앱. 매일 쓰라고 강요하는 세상의 모든 기록 앱에 반대한다.

## 왜 이 방향인가 (전략)
- 기능(AI 분류, 색 캘린더, 감정 기록)은 이미 경쟁 앱들이 다 갖고 있음 → 기능으로는 못 이김.
- 바이럴·AI는 남이 돈·기술로 따라올 수 있지만, "정체성"은 한번 선점하면 뺏기 어려움.
- "매일 성실히"를 파는 시장에서 "느슨해도 된다"를 파는 자리는 비어 있음.
- 타겟 = "기록 앱 무덤"을 가진 사람들. 죄책감 때문에 새 앱을 못 까는 사람. 대부분 기록 앱 리뷰에 "며칠 쓰다 관뒀다"가 반복됨 = 실패한 사용자가 곧 시장.

## 슬로건
- **메인: "안 써도 괜찮아요"**
- **서브: "띄엄띄엄 살아도, 기록은 남아요"**
- (메인=허락을 주고, 서브=그래도 뭔가 남는다는 작은 보상 약속. 놓지 않으면서 압박하지 않는 균형)

## 목소리 원칙 (모든 문구는 여기서 나온다)
1. 명령하지 않는다. "기록하세요" → "생각나면 던져요"
2. 평가하지 않는다. "3일 연속 실패" 같은 말은 앱에 존재하지 않음
3. 나를 낮춘다. 앱이 사용자를 위해 움직이는 톤 ("제가 정리할게요")
4. 작게 말한다. 느낌표보다 마침표. 호들갑 대신 담담함
5. 되돌아온 걸 반긴다. 떠났다 온 사람을 탓하지 않고 반가워함

## 두 가지 결과 사용 상황
**허락해주는 결 ("그래도 돼요") — 평소 기본 톤**
- 세상이 안 해주는 허락을 앱이 대신. "안 해도 돼요 / 미뤄도 돼요 / 그래도 괜찮은 거였어요"
- 허락의 레이어: ①기록 안 한 것 ②삶이 엉망인 것 ③회고 안 하는 것 모두 허락
- 가장 강력한 순간: 사용자가 스스로 탓할 때 앱이 먼저 "당신 잘못 아니에요"

**나란히 앉는 결 ("우리 그렇잖아요") — 재방문/실수/빈 나무 때만 아껴 쓰기**
- 앱이 관찰자가 아니라 같은 처지. 자기도 완벽하지 않은 척.
- "우리 원래 이렇잖아요" / "저도 정리 완벽하겐 못해요, 그래도 해볼게요" / "이번 달 우리 둘 다 좀 게을렀네요"
- 주의: 억지 친근함/느끼함 주의. 결정적 순간에만 아껴 쓸 것.

## 마이크로카피 예시 (앱 곳곳의 목소리)
- 오랜만에 열었을 때: "오랜만이에요. 그동안 뭐 있었어요?"
- 빈 나무: "아직 씨앗이에요. 급할 거 없어요."
- 기록 삭제: "지웠어요. 원래 그런 거예요."
- 알림: "3일 만에 생각났어요. 뭐 하나 던져볼래요?"
- 태그 수정: "아, 그거 아니었구나. 고쳤어요."
- 충동구매 기록: "그럴 수도 있죠. 기록해둘게요."
- 실수 반복: "또 그랬네요. 근데 뭐, 사람이 그렇죠."
- 폭식 기록 후: "요즘 좀 그랬구나. 알겠어요."
- 할 일 계속 미룸: "아직 안 했어도 괜찮아요. 여기 그대로 있어요."

## 앱스토어 소개 문구 (초안)
> **툭 — 안 써도 괜찮은 기록 앱**
> 다이어리 앱, 가계부 앱, 습관 앱... 깔았다가 며칠 만에 관둔 적 있죠?
> 툭은 매일 쓰라고 하지 않아요.
> 생각나면 그냥 아무거나 던지세요. 먹은 것, 산 것, 기분, 할 일 — 뭐든요.
> 정리는 AI가 알아서 해요. 당신은 던지기만 하면 돼요.
> 며칠 빼먹어도 잔소리 없어요. 한 달에 한 번, 당신이 던진 것들이
> 한 그루 나무로 자라나는 걸 보여드릴게요.
> 계획 없이 사는 당신에게, 딱 맞는 기록.

## 첫 화면(온보딩) 카피
> (크게) **안 써도 괜찮아요**
> (작게) 지금까지 기록 앱, 다 실패했어도. 툭은 그런 당신을 위한 거예요.
> [가입 없이 일단 던져보기]

## 지켜야 할 균형 (중요)
- "괜찮아요"가 "될 대로 되라"가 되지 않게. 따뜻하되 무책임하지 않은 선.
- 미성숙함(충동구매, 야식 등)은 긍정하되, 반복 누적으로 사용자가 스스로 힘들어하는 신호가 보이면 판단 없이 넌지시 살펴봐주는 톤으로 전환.
- "강요는 안 하지만 유혹은 한다": 안 써도 된다는 태도는 유지하되, 다시 데려오는 장치(랜덤 알림, 월말 나무 이벤트)는 따로 강하게 설계.

# [추가] 재방문(리텐션) 장치 설계 — "강요는 안 하지만 유혹은 한다"

## 원칙
안 써도 된다는 태도는 유지하되, 돌아오고 싶게 만드는 장치는 따로 강하게 설계. 강요·죄책감 없이 "유혹"만.

## 장치 1: 랜덤 알림
- 하루 최대 1회, 예측 불가한 시간 (새벽 제외). 매일 다른 랜덤 시드.
- 압박형 금지: "3일째 안 쓰셨어요" (X)
- 톤: 나란히 앉는 결. "생각난 김에 하나만 던져볼래요?" "오늘 하루 어땠어요?"
- 오래 안 왔을 때: "오랜만이에요, 별일 없었어요?" (죄책감 없이 반가움만)
- 자주 쓰면 빈도 자동 감소. 계속 무시해도 빈도 자동 감소 (스스로 눈치채는 앱).

## 장치 2: 월말 나무 이벤트 (가장 강력한 유혹)
- 매달 말 "이번 달 나무가 다 자랐어요" 알림 1회.
- 스포티파이 랩드처럼 "내 나무 보러 가는" 기다려지는 순간으로.
- 적게 쓴 사람도 부끄럽지 않게 → 앙상한 나무도 "여백이 있는 나무"로 예쁘게.
- 나무를 9:16 이미지로 공유 → 바이럴 연결.

## 장치 3: 되돌아왔을 때의 환대 (재방문 습관화의 핵심 순간)
- 오랜만(예: 7일+)에 열면 첫 화면에서 탓하지 않고 반겨주기.
- "다시 와줘서 반가워요" / "오랜만이에요. 그동안 뭐 있었어요?"
- 스트릭 끊긴 것을 오히려 반김 (스트릭 끊김을 축하하는 앱은 없음).

## 균형
- 이 셋으로 "유혹"은 강하게, 하지만 열었을 때의 경험은 여전히 "안 써도 괜찮은" 편안함 유지.

# [추가] 제품 결정 & 장기 리텐션 (빈 화면 / 숲 / 검색 / 애매한 태그 / 알림)

## 1. 첫 사용자 빈 화면 (첫날 첫인상)
- 텅 빈 게 아니라 "시작점"으로 연출. 홈: 🌱 + "첫 씨앗을 심어볼까요? 위에 뭐든 하나 던지면 시작이에요."
- 나무 탭: 흙+씨앗만 있는 화면 (자라날 자리). 압박 없이 설레게.
- 죄책감/공백 강조 금지.

## 2. 애매한 태그 처리
- 억지 분류 안 함. 확신도 낮으면 "미정(회색)"으로 두거나 그냥 메모로.
- 사용자가 원할 때만 나중에 붙임. "몰라도 괜찮다"는 태도 = 앱 정체성과 일치.
- "심심해" 같은 표현은 실제 LLM이 감정으로 분류 (키워드 매칭으론 못 잡음 → 실제 AI 필수).

## 3. 알림 시작 시점 & 톤
- 앱 깐 첫날엔 알림 없음. 3일 정도 + 기록이 좀 쌓인 뒤부터 가끔.
- 하루 최대 1회 랜덤 시간(새벽 제외). 문구 뱅크 20~30개로 시작.
- 압박형 금지, 나란히 앉는 결. "생각난 김에 하나만?" / 오래 비면 "오랜만이에요".

## 4. 장기 리텐션: 나무 → 숲 + 검색
- **숲 뷰**: 나무 탭에서 "숲 보기" 토글 → 달별 미니 나무들이 grid로 늘어섬. 1년 쓰면 12그루의 나만의 숲. 미니 나무 누르면 그 달 상세로. 다음 달 자리는 빈 화분(🪴)으로 예고.
- **검색**: 나무 탭 상단 검색바. 텍스트·태그로 전체 기록 검색 ("그때 그거 뭐였지"). P도 찾고 싶을 때가 있음. 결과 없으면 "그런 기록은 없네요. 다른 말로 찾아볼래요?"
- 숲은 "쌓임의 시각화" = 오래 쓸수록 보상이 커지는 구조 = 강력한 장기 리텐션.

## 5. P 여부 안 물어봄 (결정)
- 온보딩에서 MBTI 안 물어봄. "실패해도 괜찮은 앱"이 "P를 위한 앱"보다 넓고 강한 컨셉.
- P는 마케팅 언어로만 사용. 앱 자체는 "안 써도 괜찮은 사람 누구나"로 개방.

## 디자인 마무리 방침
- 색감·간격 등 픽셀 단위 미세조정은 Claude Code에서 실제 데이터로 진행 (지금 목업은 시드 데이터 기준).
- 지금 목업에서 확정된 것: 레이아웃, 시각화 종류(나무/색달력/파도/포스트잇/음식아이콘/숲), 라이트·다크 테마, 브랜드 보이스, 화면 흐름.

# [추가] AI 분류 프롬프트 설계 (개발 핵심 재료)

## 목표
사용자가 던진 한 줄(텍스트/음성전사)을 받아서 → 카테고리 1개 + 세부 태그 최대 2개 + (있으면) 사람 이름을 일관되게 반환. 확신 낮으면 억지 분류하지 않고 "미정".

## 입출력 형식
- 입력: 사용자 원문 텍스트. (선택) 그 사용자의 과거 수정 이력 = 개인화 사전.
- 출력: JSON. 예)
  ```json
  {
    "category": "감정",
    "subtags": ["무기력"],
    "people": [],
    "confidence": 0.82,
    "undecided": false
  }
  ```
- category는 [식단, 감정, 할일, 소비, 관계] 또는 null(미정).
- subtags는 최대 2개. category에 속한 세부 태그.
- people: 관계 맥락에서 등장한 사람(이름/호칭). 예 ["지수","팀장"].
- confidence: 0~1. 낮으면 undecided=true 로 두고 category=null 가능.

## 시스템 프롬프트 초안 (개발 시 이 방향으로)
> 너는 사용자가 하루 중 아무렇게나 남긴 짧은 기록을 분류하는 역할이야.
> 규칙:
> 1. 카테고리는 정확히 다음 다섯 중 하나이거나, 애매하면 null: 식단/감정/할일/소비/관계.
> 2. 세부 태그는 그 카테고리에 자연스럽게 속하는 것만, 최대 2개. 억지로 채우지 마.
> 3. 사람(이름/호칭: 엄마, 팀장, 지수 등)이 등장하면 people에 원문 표기 그대로 넣어.
> 4. 감정을 단정적으로 진단하지 마. "무기력"처럼 관찰 가능한 상태 태그만 붙여.
> 5. 확신이 낮으면(예: 0.6 미만) undecided=true, category=null 로 둬. 억지 분류가 오분류보다 나빠.
> 6. 출력은 지정한 JSON만. 설명 문장 금지.
> (제공되면) 아래는 이 사용자가 과거에 직접 고친 매핑이야. 우선 참고해: {개인화_사전}

## 세부 태그는 고정 목록이 아니라 "확장 가능"
- 초기엔 카테고리별 시드 태그 세트를 제공하되, 사용자 기록에서 반복되는 새 주제는 새 태그로 제안 가능.
- 새 태그 제안 시엔 사용자 승인 후 그 사용자 사전에 추가. AI가 무한정 새 태그를 만들지 않게 상한/빈도 조건.

## 개인화 학습 루프
- 사용자가 태그를 고치면 (원문 표현 → 고른 태그) 매핑을 그 사용자 사전에 저장.
- 다음 분류 때 이 사전을 프롬프트에 주입해 우선 반영. → "나를 배워가는" 경험.
- 전체 사용자의 공통 오분류 패턴은 주기적으로 모아 기본 시드 태그/규칙 개선.

## 미정(undecided) UX
- category=null이면 홈 카드에 회색 "미정" 표시하거나 태그 없이 메모로.
- 강요 없음. 사용자가 원할 때 카드 펼쳐서 직접 붙일 수 있음.
- "심심해", "그냥 그랬다" 같은 표현은 실제 LLM이면 대개 감정으로 분류됨(키워드 매칭 한계 → 실제 AI 필수).

## 비용/성능 주의
- 기록마다 LLM 1회 호출 → 사용자 수에 비례해 비용 증가. 베타에서 실제 단가 측정.
- 짧은 입력이 대부분이므로 작고 빠른 모델로 충분한지 실험. 배치/캐싱 여지 검토.

# [추가] 안전 대응 설계 (자해·위험 기록)

## 원칙
- 툭은 "판단하지 않는 앱"이지만, 자해·자살·타해 암시 등 안전 신호는 예외적으로 반드시 대응.
- 단, 놀라게 하거나 진단하듯 굴지 않음. 조용하고 담담하게, 도움 경로만 제시.

## 분류 단계에서
- AI 분류 시 안전 위험 신호를 별도 플래그로 감지(예: risk=true). 이건 카테고리 분류와 독립적으로 항상 확인.
- 위험 신호가 있으면 그 기록엔 가벼운 태그(#무기력 등)를 붙여 넘기지 말고, 안전 응답 경로로 전환.

## 대응 방식
- 판단·진단 문구 금지. "번아웃이에요" 같은 단정 금지.
- 담담하게 곁에 있어주는 톤 + 전문 도움 안내. 예: "많이 힘들었겠어요. 혼자 감당하지 않아도 돼요."
- 한국 기준 실제 창구 안내(예: 자살예방상담전화 등 최신 정보 확인 후 반영). 링크/번호는 개발 시점 최신으로 검증.
- 특정 수단·방법을 언급하거나 상세화하지 않음. 위로+연결에만 집중.

## 미성년자
- 만 14세 미만 가입 시 법정대리인 동의. 미성년 사용자 데이터 보호 강화.
- 소비·감정 데이터 민감성 고려해 연령 확인 실질적으로 작동하게.

## 반드시 전문가 자문
- 감정·안전 관련 문구와 플로우는 실제 출시 전 심리/안전 전문가 자문 필수.
- 이 문서의 안전 관련 내용은 방향 제시이며, 실제 문구·창구·정책은 전문가 검토 후 확정.
